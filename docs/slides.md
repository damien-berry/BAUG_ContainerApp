# Slides

This file is the talk note scaffold for the Brisbane Azure User Group presentation.

## Current outline

1. Home: neutral opening slide for the Build 2026 announcement and AI-native runtime discussion.
2. What Changed My Mind: the turning point where the session became an architecture discussion rather than a model showcase.
3. Problem: why runtime challenges matter more than model novelty.
4. Runtime: what an agent runtime must do in production.
5. Architecture: the sandbox and session model demonstrated at Build.
6. Demo 1 and Demo 2: live walkthrough moments showing agent behaviour and stateful interaction.
7. Sandboxes, Session Groups, Snapshots, Egress Policies and Identity: the runtime concepts that frame the real story.
8. My Take, Where I Would Use It, Trust Spectrum and Adoption Criteria: the architect's view on where ACA Sandboxes fit.
9. Recap and Resources: final takeaways and follow-up links.

## Current narrative arc

- Start neutral: the audience should believe this is a talk about AI agents and Azure Container Apps.
- Pivot reflectively: the deeper documentation reveals that the real story is runtime architecture.
- Move into practical framing: runtime, isolation, trust, state, access and governance.
- Finish with judgement: where these capabilities are useful, where they are not, and what to investigate next.
